# 대시보드 개편 개발 계획 (v3 - 최종)

## 핵심 컨셉
> 한 종목의 **일별 비중 증감 흐름 전체**를 보여줘서
> "이 종목이 계속 늘리고 있는지, 줄이다 늘렸는지" **추세를 파악**하는 대시보드

---

## 확정 사항

1. Top 20 → **Top 50** 전체 노출 ✅
2. 필터 기준: **비중(weight) 증가/감소** 기준 (기존 stock_count → weight) ✅
3. 정렬: **거래대금 > 비중 > ETF수** ✅
4. 거래대금 = **주식수 변화 × 종가** (현재 방식 유지) ✅
5. "증가주식수" 컬럼 **유지** ✅
6. 기간 내 증가일/감소일 모두 있는 종목 → **매수/매도 Top 양쪽에 다 노출** ✅
7. 날짜별 비중 변화 판단: **ETF별 개별 판단** ✅
8. 기간 선택: 1/3/5/10일 버튼 + **숫자 직접 입력** ✅
9. 날짜별 증감 컬럼: 증가/감소 **모든 날짜 변화** 표시 (증가만 X) ✅
10. 종목 클릭 드릴다운: 선택 기간 내 **날짜별 상세** (거래대금/ETF수/비중/주식수) ✅

## UI 설계

### 메인 테이블 (매수 증가 Top 50)

기간 내 거래대금 **순증가 합계**가 큰 순서로 정렬.

| # | 종목명 | ETF수 | 증가주식수 | 거래대금 | 비중변화 | 날짜별 증감 |
|---|--------|-------|-----------|---------|---------|-----------|
| 1 | 덕산하이메탈 | 5 | +1,234 | +6,456만 | +5.06 | `3/11 +3,000만` `3/12 -1,500만` `3/13 +2,000만` |
| 2 | 두산 | 3 | +800 | +4,200만 | +3.20 | `3/11 +1,000만` `3/12 +1,200만` `3/13 +2,000만` |

- **거래대금**: 기간 내 날짜별 거래대금 순증감의 합계
- **비중변화**: 기간 내 날짜별 비중 변화의 합계
- **ETF수**: 기간 내 비중이 증가한 고유 ETF 수
- **증가주식수**: 기간 내 주식수 증가분 합계
- **날짜별 증감**: 모든 날짜의 거래대금 증감 표시 (증가=빨강, 감소=파랑)

### 종목 클릭 → 드릴다운 (기존 기능 확장)

| 날짜 | 거래대금 | ETF수 | 비중변화(%p) | 주식변화 | 종가 |
|------|---------|-------|-------------|---------|------|
| 03-13 ← 03-12 | +2,000만 | 3 | +0.45 | +300 | 52,300 |
| 03-12 ← 03-11 | -1,500만 | 1 | -0.20 | -200 | 52,100 |
| 03-11 ← 03-10 | +3,000만 | 4 | +0.80 | +500 | 51,800 |
| **합계 (3일)** | **+3,500만** | **4** | **+1.05** | **+600** | |

### 매도 증가 Top 50
- 동일 구조, 거래대금 **순감소 합계**가 큰 순서로 정렬

### 기간 선택 UI
- 1일 / 3일 / 5일 / 10일 빠른 선택 버튼
- 숫자 직접 입력 가능한 input 필드

## 백엔드 변경 (`analyzer/signal.py`)

### `get_top_buy_increase` 리팩토링

**기존**: 최신일 vs N일전 두 날짜만 비교
**변경**: 기간 내 **모든 연속 날짜쌍** 비교 → 날짜별 증감 데이터 수집

```
로직:
1. 기간 내 수집 날짜 목록 조회 (예: 3일 → [03-13, 03-12, 03-11, 03-10])
2. 연속 날짜쌍별 비교 (03-13↔03-12, 03-12↔03-11, 03-11↔03-10)
3. 각 날짜쌍에서 ETF별 비중 변화 → ETF별 개별 판단
4. 종목별 집계:
   - daily_changes: [{date, trade_amount(부호있음), weight_change, stock_change, etf_count}, ...]
   - 합계: trade_amount 순증감, weight_change 순증감, stock_change 순증감
5. 기간 내 비중 증가한 날이 1일이라도 있으면 매수 Top 후보
6. 정렬: 거래대금 순증감 > 비중 순증감 > ETF수
```

**응답 구조:**
```json
{
  "stock_name": "덕산하이메탈",
  "etf_count": 5,
  "total_increase": 1234,
  "weight_change": 5.06,
  "stock_price": 52300,
  "trade_amount": 64560000,
  "daily_changes": [
    {"date": "2026-03-13", "trade_amount": 20000000, "weight_change": 0.45, "stock_change": 300, "etf_count": 3},
    {"date": "2026-03-12", "trade_amount": -15000000, "weight_change": -0.20, "stock_change": -200, "etf_count": 1},
    {"date": "2026-03-11", "trade_amount": 30000000, "weight_change": 0.80, "stock_change": 500, "etf_count": 4}
  ]
}
```

# 날짜별 비중 변화 페이지 (신규)

## 핵심 컨셉
> 특정 날짜 하루에 어떤 종목이 액티브 ETF에서 가장 많이 비중이 변했는지 한눈에 파악

## 확정 사항

1. 라우트: `/daily` ✅
2. 날짜 선택: **달력(datepicker)**으로 직접 선택, DB 수집일만 선택 가능 ✅
3. 기본 날짜: 접속일 기준 → 수집일 아니면 **가장 최근 수집일** 자동 선택 ✅
4. 비교 기준: **선택 날짜 vs 직전 수집일** (하루 변화) ✅
5. 테이블: **매수/매도 분리 없이 하나의 테이블** ✅
6. 정렬: **거래대금 양수(매수)가 위, 음수(매도)가 아래** → 거래대금 > ETF수 > 비중 ✅
7. 종목 클릭 드릴다운: **ETF별 상세** (어떤 ETF에서 비중이 변했는지) ✅
8. 드릴다운에 **보유한 모든 ETF 표시** + 보유금액(거래대금) 출력 ✅

---

## UI 설계

### 상단: 날짜 선택
- HTML5 date input (달력)
- 수집일 아닌 날짜 선택 시 → 가장 가까운 이전 수집일로 스냅
- 표시: "2026-03-13 (vs 2026-03-12)"

### 메인 테이블

| # | 종목명 | 거래대금 | ETF수 | 비중변화(%p) | 주식변화 | 종가 |
|---|--------|---------|-------|-------------|---------|------|
| 1 | 덕산하이메탈 | +2,086만 | 2 | +5.06 | +1,485 | 14,050 |
| 2 | 두산 | +6,045만 | 5 | +4.62 | +800 | 12,000 |
| ... | ... | ... | ... | ... | ... | ... |
| 48 | SK하이닉스 | -3,200만 | 3 | -1.20 | -150 | 210,000 |

- 양수(매수 증가) → 빨강(num-up), 위쪽
- 음수(매도 감소) → 파랑(num-down), 아래쪽

### 종목 클릭 → ETF별 드릴다운

| ETF명 | 보유주식수 | 보유금액 | 비중(%) | 주식변화 | 비중변화(%p) | 거래대금(변화) |
|-------|----------|---------|--------|---------|-------------|-------------|
| TIMEFOLIO 코리아밸류업 | 4,042 | 5,679만 | 4.03 | +4,042 (신규) | +4.03 | +5,679만 |
| RISE 비메모리반도체 | 1,485 | 2,086만 | 4.97 | +1,485 (신규) | +4.97 | +2,086만 |

- **보유한 모든 ETF** 표시 (변화 없어도 보유 현황 포함)
- 변화 있는 ETF는 주식변화/비중변화/거래대금 표시
- 변화 없는 ETF는 보유 현황만 표시 (변화 컬럼 `-`)

## 백엔드 (`analyzer/signal.py`)

### `get_daily_snapshot(target_date)` 신규 함수

```
로직:
1. target_date와 직전 수집일 찾기
2. 두 날짜의 모든 ETF 구성종목 조회
3. 종목별 집계: 거래대금, ETF수, 비중변화, 주식변화, 종가
4. 정렬: 거래대금 내림차순 (양수→음수)
```

응답 구조:
```json
{
  "target_date": "2026-03-13",
  "prev_date": "2026-03-12",
  "stocks": [
    {"stock_name": "덕산하이메탈", "trade_amount": 20864250, "etf_count": 2,
     "weight_change": 5.06, "stock_change": 1485, "stock_price": 14050},
    ...
  ]
}
```

### `get_stock_etf_detail(stock_name, target_date)` 신규 함수

```
로직:
1. target_date와 직전 수집일의 해당 종목 ETF별 데이터 조회
2. 모든 보유 ETF 표시 + 변화량 계산
```

응답 구조:
```json
{
  "stock_name": "덕산하이메탈",
  "target_date": "2026-03-13",
  "prev_date": "2026-03-12",
  "etfs": [
    {"etf_name": "TIMEFOLIO 코리아밸류업", "etf_code": "474590",
     "stock_count": 4042, "holding_amount": 56790100, "weight": 4.03,
     "stock_change": 4042, "weight_change": 4.03, "trade_amount": 56790100},
    ...
  ]
}
```
